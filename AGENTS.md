# AGENTS.md

## 工作方式
- 每次只解决一个明确的小任务。
- 禁止无故扫描整个项目。
- 禁止无关重构。
- 禁止一次性修改多个模块。
- 修改前先说明准备查看哪些文件，以及为什么。
- 每次最多先查看 5 个相关文件。
- 如果上下文不足，优先询问或查看项目地图文件，不要全仓搜索。
- 修改后必须给出：改了哪些文件、为什么改、如何验证、风险点。

## 代码原则
- 保持现有项目风格。
- 不随意引入新依赖。
- 不改变已有功能逻辑，除非任务明确要求。
- 优先做最小改动。
- UI 修改要保持原有视觉风格一致。

## 输出要求
- 中文说明。
- 代码改动要简洁。
- 不要写长篇无关解释。
