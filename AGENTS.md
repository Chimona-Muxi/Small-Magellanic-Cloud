# AGENTS.md

这份文件给后续维护 Small Magellanic Cloud 的代码代理看。目标很简单：少扫、少猜、少重构，把每次修改做得稳、准、可验证。

## 工作契约

- 每次只解决一个明确任务。
- 修改前先说明准备查看哪些文件，以及为什么。
- 上下文不足时，优先看 `README.md`、`STYLE_GUIDE.md` 或相关页面文件，不做无目的全仓扫描。
- 默认最多先查看 5 个相关文件；如果任务确实跨模块，再说明原因后扩大范围。
- 不回退用户已有改动，不把无关重构混进当前任务。
- 不随意引入新依赖，不改变已有功能逻辑，除非任务明确要求。

## UI 契约

- UI 修改必须先对齐 `STYLE_GUIDE.md`。
- Small Magellanic Cloud 的气质是：安静、清楚、深远、克制、有秩序。
- 该大的地方要大：主体内容、当前状态、主要操作。
- 该小的地方要小：辅助说明、低频状态、补充入口。
- 该隐身的地方要隐身：重复标题、重复按钮、纯装饰、低频设置。
- 设置、导航、卡片、棋盘、工具输入必须稳定，不因 hover、语言变化或状态变化跳动。
- 多语言文案优先进入 `public/preferences.mjs`，不要在页面里散落硬编码文本。

## 代码契约

- 保持现有项目风格和文件边界。
- 首页、模块页、学习工具页共享 `public/styles.css` 与 `public/site.mjs`。
- 棋类游戏页有各自独立样式；修改全站设置体验时要检查它们是否有重复规则。
- `server.mjs` 同时承载静态服务、棋类联机和 p-adic API，改动前要明确影响面。
- 工具脚本和安装器属于发布资源，除非任务需要，不做格式化或重排。

## 验证契约

- JS 改动至少跑 `node --check`。
- UI 改动尽量启动本地服务，并用浏览器实际验证桌面视口。
- 设置面板改动至少验证首页、学习工具页和一个棋类页。
- 语言改动至少验证一个非中文语言，包含标题、按钮、状态和可访问标签。
- 修改后汇报：改了哪些文件、为什么改、如何验证、剩余风险。

## 提交与推送契约

- 完成项目后通过 GitHub Desktop 提交到仓库：[Chimona-Muxi/Small-Magellanic-Cloud.git](https://github.com/Chimona-Muxi/Small-Magellanic-Cloud.git)。

## 输出契约

- 用中文说明。
- 先说结果，再说验证。
- 文件引用要具体。
- 不写长篇无关解释。
- 有风险就明说，不用漂亮话盖过去。
